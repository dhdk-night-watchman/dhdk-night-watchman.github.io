# AI Use Statement

## Project Context

This document records the use of artificial intelligence assistance during the development of **The Night Watchman**, a university group project for an interactive digital heritage prototype. The project remains a human-authored academic submission. AI assistance was used as a support tool for review, implementation planning and selected incremental improvements.

## AI Tools Used

AI assistance was provided by OpenAI Codex through the Codex desktop environment. The tool was used to inspect the existing repository, compare the implementation against the design brief, and assist with targeted improvements requested by the project team.

## Purpose of AI Assistance

AI assistance was used for the following purposes:

- Reviewing the existing HTML, CSS and JavaScript architecture.
- Identifying accessibility, user experience and digital heritage improvement opportunities.
- Supporting the implementation of incremental accessibility enhancements.
- Helping structure museum-style metadata for the existing object study sections.
- Drafting documentation suitable for project review and university submission.

AI was not used to replace the original concept, redesign the game, introduce a new gameplay model, or generate a separate project architecture.

## Human Review Process

All AI-assisted suggestions and code changes were reviewed in relation to the original project goals, the official design brief and the existing implementation. The human project member retained responsibility for deciding which changes were appropriate for the submission.

The review process focused on preserving:

- The existing game flow.
- The Night Watchman narrator and tone.
- The current visual identity.
- The eight-object Rijksmuseum structure.
- The forgiving hint and study-or-skip learning loop.

## Modifications Performed After AI Assistance

The AI-assisted development work was limited to incremental improvements:

- Added accessibility support through semantic regions, a skip link, visible focus states and live announcements.
- Added structured Digital Heritage metadata to the study section for each museum object.
- Added official Rijksmuseum object links in a small Learn More section.
- Added concise educational introductions for each gallery.
- Improved progress feedback through completed, remaining and overall progress indicators.
- Improved mobile layout for artwork frames, buttons and spacing.
- Expanded project documentation for maintainability and academic transparency.

No backend, database, framework, new gameplay system or major redesign was introduced.

## Academic Integrity Statement

The use of AI in this project was assistive and transparent. The final submission should be understood as a human-directed project in which AI helped with review, drafting and implementation support. The project team remains responsible for the design decisions, final content, testing and academic claims.
